Test Management User Guide

A clear, step-by-step guide to uploading and managing your examination tests.

Part 1: Uploading New Tests (Upload Tests Page)

The **Upload Tests** page offers three tabs, each providing a way to submit test data. Choose the method that best suits your existing data format.

üìù Method 1: Create with Form

1

Configure & Generate

  • Fill in **Test Title**, **Exam Type**, **Syllabus Type**, and **Negative Marks**.
  • Select **Subjects**, define the **Number of Questions** and **Marks per Question** for each.
  • Click **Generate Test Form**.
2

Enter Question Data

Complete the fields for the entire test, including **Description**, **Duration**, **Question Text**, **Subject**, **Topic**, **Solution**, and **Correct Answer** (MCQ option or NVQ number).

Tip: Image Upload

For diagrams, use the **Upload Qs. Image** and **Upload Sol. Image** buttons *within each question block*. The system handles the hosting and provides the URL automatically.

üìä Method 2: Upload from Excel (For Bulk Upload)

This method is ideal for bulk imports. Your spreadsheet must strictly follow the required template format.

1

Prepare & Configure

  • Click **Download Excel Template** to get the exact required column structure.
  • Fill the template carefully, ensuring column headers match.
  • Fill in the metadata on the portal (**Test Title**, **Exam Type**, **Subjects**, etc.).
2

Process & Review

  • Upload your prepared Excel file (.xlsx) and click **Process Excel & Preview Form**.
  • The system will parse the data and automatically switch you to the **Create with Form** tab, populating all fields.
  • **Review all questions** for correct parsing, especially complex text or images.

üíª Method 3: Upload from JSON (For Programmatic Use)

This method requires advanced knowledge of the system's data schema and is best used for direct data migration or programmatic imports.

1

Format Data

  • Ensure your data strictly matches the **Required JSON Format** shown below the input box.
  • The JSON must be a single object containing all test metadata and the `questions_data` array.
2

Paste & Upload

  • Paste the entire valid JSON object into the text area.
  • Click **Upload JSON**. The test is uploaded directly to the database without a review step.

Warning: Data Validation

Errors in the JSON structure (missing fields, incorrect types) will result in an upload failure.

3

Final Upload (Applies to Form & Excel Review)

Once satisfied, click the **Upload Test to Database** button. The system saves the full test data and its metadata to the portal's database.

Crucial Warning: Test ID

The **Test Title** is used to generate a unique ID. If a test with the same ID already exists, the system will ask for confirmation to **overwrite** the previous version. Be careful when overwriting live tests!

Part 2: Managing and Editing Tests (Manage Tests Page)

The **Manage Tests** page provides a centralized list of all uploaded tests and allows you to control their availability and content.

A

Control Availability

The **Available** column contains a toggle switch for every test. This is the **primary control** to release or hide a test from student view.

  • **Green Toggle (On/Checked):** Test is visible and accessible to students.
  • **Gray Toggle (Off/Unchecked):** Test is hidden and inaccessible to students.

B

Edit Test Content

Click the **Edit** button next to any test to access the full edit form. This form mirrors the "Create with Form" upload process and is pre-populated with the existing test data.

  • You can modify test details (Title, Duration, Marks) and the content of individual questions (Text, Options, Correct Answer, Solutions).
  • Use the **+ Add Question** button to add new questions or the **Delete Question** button to remove existing ones.

Pro Tip: Image Management

If you upload a new image while editing, the old image URL is replaced. If you change the test title while editing, the new image upload path will still use the **original Test ID** to keep related files organized.

C

Save Changes

After making any modifications, click the **Save Changes** button. Your updates are instantly applied to the database, ensuring students see the correct, updated version immediately (if the test is currently **Available**).
